---
title: Comments API
description: Public API endpoints for managing and retrieving comments and replies
---

The Comments API provides endpoints to list, create, and manage comments and replies on posts and pages.

## List Comments

Retrieves a paginated list of comments for a specific subject (post, page, etc.).

<CodeGroup>
```bash cURL
curl -X GET "http://localhost:8091/apis/api.halo.run/v1alpha1/comments?kind=Post&version=v1alpha1&name=post-name&page=0&size=10" \
  -H "Accept: application/json"
```
</CodeGroup>

### Query Parameters

<ParamField query="kind" type="string" required>
  The comment subject kind (e.g., `Post`, `SinglePage`).
</ParamField>

<ParamField query="version" type="string" required>
  The comment subject version (e.g., `v1alpha1`).
</ParamField>

<ParamField query="name" type="string" required>
  The comment subject name.
</ParamField>

<ParamField query="group" type="string">
  The comment subject group.
</ParamField>

<ParamField query="page" type="integer" default="0">
  Page number. Default is 0.
</ParamField>

<ParamField query="size" type="integer" default="0">
  Size number. Default is 0.
</ParamField>

<ParamField query="sort" type="array">
  Sorting criteria in the format: `property,(asc|desc)`. Default sort order is ascending. Multiple sort criteria are supported.
</ParamField>

<ParamField query="withReplies" type="boolean" default="false">
  Whether to include replies. Default is false.
</ParamField>

<ParamField query="replySize" type="integer" default="10">
  Reply size of the comment. Only works when `withReplies` is true.
</ParamField>

  </ResponseField>
  
  <ResponseField name="owner" type="OwnerInfo" required>
    Comment owner details.
    
    <ResponseField name="kind" type="string">

      Owner name.
    </ResponseField>

## Create Comment

Creates a new comment on a post or page.

<CodeGroup>
```bash cURL
curl -X POST "http://localhost:8091/apis/api.halo.run/v1alpha1/comments" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{
    "subjectRef": {
      "group": "content.halo.run",
      "version": "v1alpha1",
      "kind": "Post",
      "name": "post-name"
    },
    "raw": "This is a comment",
    "content": "<p>This is a comment</p>",
    "allowNotification": true
  }'
```
</CodeGroup>

### Request Body

<ParamField body="subjectRef" type="Ref" required>
  Reference to the subject (post, page, etc.) being commented on.
  
  <ParamField body="group" type="string" required>
    Extension group (e.g., `content.halo.run`).
  </ParamField>
  
  <ParamField body="version" type="string">
    Extension version.
  </ParamField>
  
  <ParamField body="kind" type="string" required>
    Extension kind (e.g., `Post`, `SinglePage`).
  </ParamField>
  
  <ParamField body="name" type="string" required>
    Extension name.
  </ParamField>
</ParamField>

<ParamField body="raw" type="string" required>
  Raw comment content (markdown/plain text).
</ParamField>

<ParamField body="content" type="string" required>
  Rendered comment content (HTML).
</ParamField>

<ParamField body="allowNotification" type="boolean" default="false">
  Whether to allow notifications for this comment.
</ParamField>

<ParamField body="hidden" type="boolean" default="false">
  Whether the comment should be hidden.
</ParamField>

<ParamField body="owner" type="CommentEmailOwner">
  Comment owner information (for non-logged-in users).
  
  <ParamField body="displayName" type="string">
    Display name.
  </ParamField>
  
  <ParamField body="email" type="string">
    Email address.
  </ParamField>
  
  <ParamField body="website" type="string">
    Website URL.
  </ParamField>
  
  <ParamField body="avatar" type="string">
    Avatar URL.
  </ParamField>
</ParamField>

### Response

Returns a `Comment` object representing the created comment.

## Create Reply

Creates a new reply to a comment.

<CodeGroup>
```bash cURL
curl -X POST "http://localhost:8091/apis/api.halo.run/v1alpha1/comments/{name}/reply" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{
    "raw": "This is a reply",
    "content": "<p>This is a reply</p>",
    "allowNotification": true
  }'
```
</CodeGroup>

### Path Parameters

<ParamField path="name" type="string" required>
  Comment name (unique identifier) to reply to.
</ParamField>

### Request Body

<ParamField body="raw" type="string" required>
  Raw reply content (markdown/plain text).
</ParamField>

<ParamField body="content" type="string" required>
  Rendered reply content (HTML).
</ParamField>

<ParamField body="allowNotification" type="boolean" default="false">
  Whether to allow notifications for this reply.
</ParamField>

<ParamField body="hidden" type="boolean" default="false">
  Whether the reply should be hidden.
</ParamField>

<ParamField body="quoteReply" type="string">
  Name of the reply being quoted.
</ParamField>

<ParamField body="owner" type="CommentEmailOwner">
  Reply owner information (for non-logged-in users).
</ParamField>

### Response

Returns a `Reply` object representing the created reply.
